## 1\. `new`의 진실하지만 단순한 개념 🏗️

### 실제 생활 비유: 아파트 짓고 이사하기
    
    
    1. Blueprint = Class
    2. Actual Apartment = Object
    3. `new` = Construction Company
    
    
    class House {    // Blueprint
        int rooms;
        string address;
    };
    
    House* myHome = new House();  // Build an actual apartment

#### `new`는 무엇을 할까요?

  1. 메모리를 **할당** 합니다 (토지를 구입합니다)
  2. 객체를 생성합니다 (아파트를 건축합니다)
  3. 생성자를 호출합니다 (유틸리티와 인테리어를 설치합니다)
  4. 메모리의 주소를 반환합니다 (입주 주소를 제공합니다)

## 2\. 생성자의 진정한 의미 🔨

### 실제 생활 비유: 이사 후 정리
    
    
    1. The house is built (object created)
    2. What comes next?
       - Connect electricity
       - Connect water supply
       - Install gas lines
       - Arrange basic furniture
       -> These "initial setups" are handled by the constructor!
    
    
    class House {
    private:
        string address;
        bool hasElectricity;
        bool hasWater;
    
    public:
        // Default Constructor (an empty house)
        House() {
            hasElectricity = false;
            hasWater = false;
        }
    
        // Custom Constructor (a fully equipped house)
        House(string addr) : address(addr) {
            hasElectricity = true;
            hasWater = true;
        }
    };
    
    int main() {
        House emptyHouse;              // A basic house
        House readyHouse("Seoul");    // A move-in ready house
    }

## 3\. `static`의 진정한 의미와 `private`을 사용하는 이유 🌍

### 실제 생활 비유: 아파트 단지의 공용 시설
    
    
    1. Things that each house (object) has:
       - Rooms
       - Kitchen
       - Bathroom
    
    2. Things shared by the entire complex (class):
       - Swimming pool
       - Parking lot
       - Playground

#### 왜 `static`은 "정적"일까요?

  * 메모리 위치가 고정되어 있습니다(이동하지 않음)

  * 프로그램 시작 시 생성되어 종료될 때까지 존재합니다

  * 모든 객체가 동일한 `static` 멤버를 공유합니다
        
        class ApartmentComplex {
        private:
          // Per-house attributes
          int roomNumber;
        
          // Shared attributes
          static int totalResidents;     // Total number of residents
          static Pool* sharedPool;       // Shared swimming pool
        




public: static void usePool() { if (sharedPool->isClosed()) cout << "The pool is closed." << endl; } };
    
    
    #### 왜 `static` 멤버에 `private`을 사용하나요?
    
    ```cpp
    class Bank {
    public:
        static int totalMoney;  // Public static member...
    };
    
    int main() {
        // Anyone can directly modify it! (Very dangerous)
        Bank::totalMoney = -999999;
    }
    
    // Instead, we should design it like this:
    class Bank {
    private:
        static int totalMoney;
    public:
        static void deposit(int amount) {
            if (amount > 0)  // Validate first
                totalMoney += amount;
        }
    };

## 모든 것을 하나로 모으기 🔄
    
    
    // Apartment Complex Management System
    class ApartmentComplex {
    private:
        // 1. `static` Members (Shared Facilities)
        static ApartmentComplex* instance;  // Management office
        static int totalUnits;              // Total number of units
    
        // 2. Non-static Members (Per Unit)
        vector<string> residents;
    
        // 3. Constructor (Initial Setup)
        ApartmentComplex() {
            cout << "Management office created." << endl;
        }
    
    public:
        // 4. Restrict `new` (Only one management office allowed)
        static ApartmentComplex* getOffice() {
            if (instance == nullptr) {
                instance = new ApartmentComplex();
            }
            return instance;
        }
    
        void addResident(string name) {
            residents.push_back(name);
            totalUnits++;
        }
    };
    
    int main() {
        // Visit the management office
        ApartmentComplex* office1 = ApartmentComplex::getOffice();
        ApartmentComplex* office2 = ApartmentComplex::getOffice();
    
        // Both point to the same office!
        office1->addResident("Kim");
        office2->addResident("Lee");
    }

### 모든 것이 연결되는 방식

  1. `new`로 객체를 생성할 때,
  2. 생성자가 초기 상태를 설정하고,
  3. `static` 멤버는 모든 객체에서 공유되며,
  4. `private`는 민감한 데이터가 보호되도록 보장합니다.

초심으로 돌아가서 나.. 소스파일이 어떻게 "패키지관리자 없이" 깔려서 install되는지 ARABOZA in Linux

#### 1\. 소스 파일로 설치하는 이유

**최신 버전 사용** : 패키지 관리자가 제공하지 않는 최신 버전을 설치하고 싶을 때. **커스터마이징** : 소프트웨어를 특정 환경에 맞게 수정하거나 설정을 변경하고 싶을 때. **패키지가 없는 경우** : 패키지 관리자(예: apt, yum)를 통해 설치할 수 없는 소프트웨어를 설치할 때.

* * *

![](https://velog.velcdn.com/images/jammy0903/post/760c19f6-075d-4d17-9c00-57fc7ba94573/image.png)

우리는 GNU Hello 라는 아주 간단한 프로그램을 다운받아서 디렉토리구조랑 코드를 확인해볼거에요
    
    
    tar -xvzf hello-2.12.tar.gz
    

이 명령어는 압축파일 해제하는 명령어인데, -xvzf 를 딥하게 설명하겠습니다.

  1. tar : 왜 tar을 사용할까? 궁금해본 적 없으신가요? tar가 개쩌는 이유는 파일 권한, 소유권, 심볼릭 링크, 특수 장치 파일 등 유닉스/리눅스 파일 시스템의 메타데이터를 완벽하게 보존합니다. 물론 크기도 개쩔게 크지만요. 대신 zip은 이러한 정보를 완전히 보존하지 못합니다.
  2. .tar.gz와 같이 확장자가 두 개인 이유는 두 가지 다른 처리 과정을 나타내기 때문( tar 노압축 -> gzip압축) 아 너무 실용적이다!



tar -xvzf의 각 옵션의 의미

  * -x : extract 압축폴더에서 꺼내와라
  * -v : verbose 자세히 출력해라
  * -z : gzip 압축풀기 .gzip푸는 유일키
  * -f [ 이름 ] : file이름 지정하는 옵션

네 ls -l하니까 이렇게 많이 주르륵 뜨네요 ![](https://velog.velcdn.com/images/jammy0903/post/b38862be-1a24-4b1c-9383-009a9e54c924/image.png)

집중해야할 디렉토리/파일 은 뭘까?

  1. src/ - 실제 프로그램의 소스 코드가 있는 가장 중요한 디렉토리
  2. configure 스크립트 - 시스템에 맞게 빌드 환경을 설정하는 실행 파일입니다. 이것이 바로 소스코드 설치의 시작점
  3. configure.ac - Autoconf 도구를 위한 설정 파일로, configure 스크립트 생성에 사용됨
  4. Makefile.am과 Makefile.in - Automake 도구에 사용되는 파일들로, 최종적으로 make 명령에 사용될 Makefile을 만드는 데 필요함



* * *

사실 딱히 관심안둬도 되는 친구들 5\. lib/ - 라이브러리 디렉토리 6\. doc/: 프로그램 문서 7\. man/: 매뉴얼 페이지 8\. po/: 다국어 지원을 위한 번역 파일들 9\. tests/: 프로그램 테스트 코드 10\. build-aux/: 빌드 시스템에 필요한 보조 스크립트들 11\. m4/: 매크로 정의 파일들 다른것들은 다른사람한테 물어보시길

## 이 프로그램을 패키지 관리자 없이 설치하려면 어떤 명령어를 순서대로 써야할까?
    
    
    ./configure
    make
    make install

왜? 환경설정 -> makfile토해내 -> 그걸로 make로 컴파일해 .c .o -> 이제 컴파일했으면 설치해! 

configure파일부터 뜯어볼까유?
    
    
    AC_CONFIG_AUX_DIR([build-aux])
    AM_INIT_AUTOMAKE([
        1.11.1
        readme-alpha
        subdir-objects
    ])
    
    AC_PREREQ([2.62])
    
    AC_CONFIG_HEADERS([config.h:config.in])dnl Keep filename to 8.3 for MS-DOS.
    AC_CONFIG_SRCDIR([src/hello.c])
    AC_PROG_CC
    
    gl_EARLY
    gl_INIT
    
    AC_DEFINE([COPYRIGHT_YEAR], [m4_esyscmd([date +%Y])], [year in copyright message])
    
    AM_CONDITIONAL([BUILD_FROM_GIT], [test -d "$srcdir/.git"])
    AM_CONDITIONAL([GIT_CROSS_COMPILING],
                   [test -d "$srcdir/.git" && test $cross_compiling = yes])
    
    AS_IF([test $cross_compiling = no], [
      AM_MISSING_PROG([HELP2MAN], [help2man])
    ], [
      HELP2MAN=:
    ])
    
    dnl i18n support from GNU gettext.
    AM_GNU_GETTEXT_VERSION([0.18.1])
    AM_GNU_GETTEXT([external])
    
    AC_CONFIG_FILES([Makefile
               po/Makefile.in])
    AC_OUTPUT
